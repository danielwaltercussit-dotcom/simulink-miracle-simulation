# Snapshot Auditor

Use this skill after `ai-in-loop` creates an S10 snapshot, or whenever a user
asks whether an exported model package is complete enough to hand off.

Primary helper:

```matlab
cd("C:\Users\jonas\Desktop\simulink_agent_v1")
init_simulink_agent_project
addpath("scripts/loop")

r = ai_in_loop_audit_snapshot(pwd, ...
    "nebus39_dfig2_weakgrid_v0", ...
    fullfile(getenv("USERPROFILE"), "Desktop", ...
    "AI summary of simulation models", "nebus39_dfig2_weakgrid_v0"));
assert(r.passed)
```

## Workflow

1. Locate the snapshot directory:
   `AI summary of simulation models/<model>/` unless the caller provides a
   custom `snapshot_root`.
2. Run `scripts/loop/ai_in_loop_audit_snapshot.m`.
3. Re-read the generated audit report before referencing the snapshot in chat:
   `build/reports/snapshots/<model>_snapshot_audit.md`.
4. Treat a failed audit as a report/package failure, not as a model dynamics
   failure. Route model behavior problems to `simulink-model-verification`,
   `diagnostic-plotting`, or `multitimescale-analysis`.

Read `references/snapshot-contract.md` before changing required files, warning
policy, or S10/S10B stage names.

## Required Evidence

The auditor expects:

- generated `.slx` model copied into the snapshot directory
- copied case spec named `case_<model>.yaml`
- copied build script named `build_<model>.m`
- `snapshot_manifest.json`
- `README.md`
- `latest_loop_status.json` with `passed=true`
- at least one markdown report copied from the project or loop iteration

Optional evidence should be preserved when present:

- `.slxc` cache
- model verification, spec validation, adapter contract, layout quality, tuning,
  sltest, Model Advisor, and top screenshot artifacts
- diagnostic plot indexes or manifests generated by `diagnostic-plotting`

## Routing

- Use `ai-in-loop` to create the snapshot.
- Use this skill to decide whether the copied snapshot package is complete.
- Use `diagnostic-plotting` when the snapshot needs waveform figures.
- Use `simulink-model-quality-layout` or `simulink-model-verification` when the
  audit fails because upstream reports are missing.
